## Getting started

**Do I need to install anything to read this site?**
No. It's a normal website — open the link in any browser.

**Do I need to install anything to edit it?**
For the Hub (this site), no — you can edit right in your browser. Only the advanced
"developer path" for the Catalog needs software installed. See
[How to Contribute]({{ '/info/contributing.html' | relative_url }}).

**Do I need to know how to code?**
No. Hub pages use **Markdown**, which is just text with a few simple symbols (like `**bold**`).
You can learn the basics in a few minutes.

**How do I get permission to edit?**
Ask your team lead to add you to the `csu-advanced-utilities-tech` GitHub organization.

## Making changes

**Will my edit appear immediately?**
Almost — once a change is saved (and approved, if it's a pull request), the site rebuilds
and publishes in about a minute or two.

**Can I break the site or lose someone's work?**
It's very hard to do permanent damage. Every version of every file is saved forever, so any
change can be undone. If you're nervous about a big edit, use a **pull request** so a
teammate reviews it before it goes live.

**What if two of us edit the same page?**
GitHub handles this gracefully. If two edits truly conflict, it flags the spot and asks
someone to pick the right version — nothing is silently overwritten.

**I don't want to edit files myself. Can I still help?**
Yes — open an **Issue** describing what should change, and someone will action it.

**What's a "pull request"?**
A request to merge your proposed changes into the live site, so they can be reviewed first.
See the [Glossary]({{ '/info/glossary.html' | relative_url }}).

## About the setup

**Why are we using GitHub instead of a shared drive or Word docs?**
One source of truth, a complete and reversible history of every change, safe review before
publishing, and a single live website everyone can read. A shared drive gives you none of
those.

**What's the difference between the Hub and the Catalog?**
The **Hub** (this site) documents *how we govern the data* — the rules, roles, and processes.
The **[Catalog](https://csu-advanced-utilities-tech.github.io/command-center-data-catalog/)**
documents *what data exists* — the tables and columns. They're separate sites that link to
each other.

**Where do the Catalog's descriptions come from?**
From spreadsheets (CSV files) we maintain. A small program turns those into the catalog web
pages, so we keep the *data* up to date rather than editing pages by hand.

## Privacy & safety

**Is this site public?**
Yes — these repositories are currently public on the internet, which is what lets the free
website hosting work. Treat everything here as publicly visible.

**So what should I NOT put here?**
Never paste **real customer data, account numbers, passwords, API keys, or sensitive PII**
into these pages. This site documents *how* we handle such data (policies, rules, structure),
not the data itself. If something sensitive needs to be referenced, link to where it's stored
securely instead.

**Can we make it private later?**
Yes. If we move to a GitHub plan that supports private Pages, the repos can be made private
without losing any content or history.

## Getting help

**Something's broken or I'm stuck — what do I do?**
Open an Issue on the repository, or ask whoever maintains the site. No question is too basic;
this portal exists precisely so everyone can participate.
